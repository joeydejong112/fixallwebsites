# Phase 2.5 â€” Best-in-Class Scan Engine

> Spec for expanding ScanPulse from 12 checks across 3 pillars to 84 checks across 6 categories.

## Decisions

| Decision | Choice | Reason |
|----------|--------|--------|
| External APIs | PageSpeed Insights + Green Web Foundation only | PSI is the only way to get real CWV field data; GWF is a free single-call check. Everything else self-contained. |
| Architecture | One file per pillar under `convex/checks/` | Keeps each file under 200 lines. Orchestrator fetches once, fans out. |
| Schema migration | Additive â€” new optional fields | Zero breaking changes to existing scans or UI. |
| AI recommendations | Static pattern matching + expanded FIX_SNIPPETS | No LLM cost. Tech detection + platform-keyed snippets covers 90%. LLM-generated recs deferred to Phase 3 Pro. |
| DNS & Email / Trust scoring | Displayed separately, not in overall score | Clients shouldn't get a 60 because they lack DKIM. These are bonus categories. |

### Shared Interface

```ts
// convex/checks/types.ts
export interface ScanIssue {
  pillar: string
  severity: 'critical' | 'warning' | 'pass'
  title: string
  description: string
}

export interface PillarResult {
  score: number
  issues: ScanIssue[]
}

export interface TlsInfo {
  validTo: string | null      // ISO date
  protocol: string | null     // "TLSv1.3"
  issuer: string | null
}

export interface CwvData {
  lcp: number | null           // milliseconds
  inp: number | null           // milliseconds
  cls: number | null           // unitless
  available: boolean           // false if PSI API fails or no CrUX data
}

export interface DnsData {
  hasA: boolean
  hasAAAA: boolean
  mx: string[]
  spf: string | null
  dmarc: string | null
  dkimFound: string[]          // selectors that resolved
  dnssecValid: boolean | null  // null if check failed
  resolveTimeMs: number
}
```

### Orchestrator Flow

```
scanAction.ts handler:

1. fetch(url) â†’ headers, html, statusCode, responseTime
   - 15s timeout, ScanPulse/1.0 User-Agent

2. IN PARALLEL (Promise.allSettled):
   a. PageSpeed Insights API â†’ CwvData
   b. tls.connect(hostname, 443) â†’ TlsInfo
   c. DNS lookups (all parallel internally):
      - resolve A, AAAA, MX, TXT (for SPF)
      - resolveTxt('_dmarc.' + domain)
      - 8 DKIM selector probes
      - DoH query for DNSSEC AD flag
   d. RDAP fetch â†’ domain expiry date
   e. Green Web Foundation API â†’ boolean
   f. HEAD requests (parallel):
      - /.env, /.git/HEAD, /phpinfo.php
      - /robots.txt
      - /sitemap.xml
      - /.well-known/gpc.json
   g. GET nonexistent path â†’ custom 404 check
   h. GET www vs non-www variant â†’ redirect check

3. Fan out to pillar modules (synchronous, pure functions):
   - security(headers, html, url, tlsInfo, exposureResults)
   - performance(headers, html, url, cwvData, greenHosting)
   - seo(headers, html, url, robotsTxt, sitemap, wwwCheck)
   - accessibility(html)
   - dns(dnsData, domainExpiry)
   - trust(html, headers, gpcResult, custom404Result)
   - techDetect(headers, html)

4. Compute scores:
   - Each pillar: (passing / total) * 100
   - Overall: avg(security, performance, seo, accessibility)
   - DNS & Trust: separate scores, not in overall

5. Save via updateScan mutation
```

Every parallel call uses `Promise.allSettled` so failures are graceful â€” if PageSpeed API is down, performance scores from the remaining checks. If TLS connect fails, skip cert expiry check. No single failure blocks the scan.

## Scoring Model

**Core pillars (in overall score):**
- Security: (passing / 21) * 100
- Performance: (passing / total available) * 100 â€” total varies based on PSI availability
- SEO: (passing / 19) * 100
- Accessibility: (passing / 12) * 100
- **Overall = avg(security, performance, seo, accessibility)**

**Bonus categories (displayed separately):**
- DNS & Email: (passing / 8) * 100
- Trust & Compliance: (passing / 6) * 100

Severity-weighted scoring considered and rejected â€” simple pass/total is transparent and easy to explain. A check either passes or it doesn't.

**Legacy compatibility:** Scans without accessibility/dns/trust scores display and average only the fields present.
